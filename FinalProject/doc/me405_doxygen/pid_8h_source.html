32 /** \brief This class runs a 16-bit fixed point PID
33  * \details The PID controller can run in P, PI, PD, PID, and manual modes. When the integral action is
34  * used, a fourth gain, Kw can be used to help eliminate windup problems. To help with reset
35  * spikes, the difference computed for the derivative action is on the input value, not the error.
36  * The block diagram below shows the entirety of the controller and it's context with respect to
37  * the plant. The @c input and @c output attributes are from the @e controller's perspective, not
38  * the plant. That is, @c input is the output of the plant into the controller and @c output is the
39  * input to the plant from the controller.\n
40  * \verbatim
41  PID CONTROLLER w/ ANTI-WINDUP
42  ##################################################################
43  # +------+ #
44  # | | #
45  # +-----+ Kw <--------------------+ #
46  # | | | +--+--+ #
47  # (-)| +------+ (+)| |(-) #
48  # +--v--+ +------+ +---> Σ <---+ #
49  # (+)| | | | | | | | #
50  # +--> Σ +--> Ki/s +-------+ | +-----+ | #
51  # | | | | | | | | #
52  # | +-----+ +------+ |(+) | MAX | #
53  # +-----+ | +------+ +--v--+ | +-------+ | # +------ +
54  SETPOINT # (+)| | | | | (+)| | | | -- | | # OUTPUT | |
55  ---------->-----> Σ +--+-----------> Kp +----> Σ +--+--> / +--+--+--------> PLANT +----+------>
56  # | | | | | | | -- | # | | |
57  # +--^--+ +------+ +--^--+ +-------+ # +------ + |
58  # (-)| +------+ (+)| MIN # |
59  INPUT # | | | | # |
60  +--------->------- +-----------------> Kd*s +-------+ # |
61  | # | | # |
62  | # +------+ # |
63  | ################################################################## |
64  +------------------------------------------------------------------------------------------------+
65 \endverbatim
66  * To use the pid controller you must first call the constructor with a pointer
67  * to a serial object.\n
68  * @c pid* @c motor_pid @c = @c new @c pid(p_serial); \n
69  * The system parameters must then be set. This can be done using the individual set function for
70  * each parameter, or all at once using a configuration struct. For example, we can set the
71  * controller to PI mode with gains Ki=5, Kp=2, and Kw=1. Remember that the gains are entered as
72  * 1024 times the desired gain to increase resolution. Additionally, we will set the output
73  * saturator values to +/- 1600. \n
74  * @c motor_pid->set_config(pid::config_t{PI,5120,2048,0,1024,-1600,1600});\n
75  * Now, to run the PID, we must call compute() inside a loop (probably a task) running at a fixed interval.
76  * Changing the delay of the task will change the effect of the Ki and Kd gains.
77  * Suppose we have three @c int16_t shares @c motor_speed, @c motor_setpoint, and @c motor_power.
78  * We can easily run the PID in one line.\n
79  * @c motor_power->put(motor_pid->compute(motor_speed->get(),motor_setpoint->get()));\n
80  */
